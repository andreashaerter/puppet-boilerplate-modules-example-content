module: filezilla

Manages FileZilla, a feature-rich, cross-platform Open Source file transfer client. It comes with support for FTP, SSH File Transfer Protocol (SFTP) and FTP over SSL/TLS (FTPS).

This module does not manage FileZilla server (which is a FTP server for windows by the same developers).

Have a look at the main module class (init.pp) to see what this module does on a node plus usage examples.

Security note

Like many other applications, FileZilla stores all handled login data and credentials as plain text files. Have a look at the following directory:

  • ~/.filezilla/ on Unix-like systems.
  • %AppData%\FileZilla\ on MS Windows (%AppData% is a Windows environment variable, pointing to the current application data dir of your account. Try to access %AppData%\FileZilla\ directly via Windows Explorer and you‘ll see how it works).

You will find recentservers.xml and sitemanager.xml there after using the program to log into a system. This is expected behavior because it is your operating system‘s responsibility too keep these files save. But keep it in mind if you are using a non-encrypted and/or multi-user systems.
